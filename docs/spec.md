# PNGKing 产品规格（SDD）

## 1. 产品目标

PNGKing 是一个完全在浏览器本地运行的批量图片优化工具。用户可拖放或选择多张图片，应用会对普通 PNG 执行自适应颜色量化与数据流重压缩，并移除非必要元数据，最后下载单个结果或 ZIP 压缩包。

## 2. 压缩边界

- **支持格式：** PNG、JPEG、WebP、GIF。
- **保证：** 不改变分辨率和文件格式；动画 PNG 不做颜色量化，始终保留动画帧和原始像素。普通 RGB/RGBA PNG 可能合并近似颜色。
- **优化方式：** 普通 8-bit RGB/RGBA PNG 会根据 1–9 级压缩强度（默认 6）选择调色板颜色数量，并与无损重过滤、最高级别 DEFLATE 结果比较；目标大小模式会枚举多个调色板候选，选择不超过目标且画质最高的结果。所有格式都会移除注释、EXIF、XMP、编辑器信息等可选元数据，同时保留透明度、动画和关键结构数据。
- **限制：** 已经充分压缩且没有冗余元数据的图片可能不会变小；若优化结果不小于原文件，应用保留原文件，节省量显示为 0 B。JPEG、WebP 和 GIF 当前仅清理安全元数据，因此部分目标体积可能无法达到；此时应用终止该图片的压缩并提示最小候选体积。
- **隐私：** 文件不上传服务器，所有读取、处理和打包均发生在当前浏览器中。

## 3. 用户故事与验收标准

### US-01 批量导入

作为用户，我可以拖放图片或打开文件选择器一次导入多张图片。

- 接受 PNG/JPEG/WebP/GIF。
- 拒绝不支持的格式并显示清晰提示。
- 相同文件名、大小、最后修改时间的文件不会重复加入。
- 导入成功后进入编辑页并自动开始优化。

### US-02 查看处理结果

作为用户，我可以看到每个文件的预览、格式、原始大小、优化后大小、节省比例与状态。

- 状态包括：等待、处理中、完成、失败。
- 顶部汇总文件数量、原始体积、优化后体积与节省比例。
- 可按名称、体积、节省比例排序。

### US-03 管理队列

作为用户，我可以删除单个文件、清空队列，并撤销或重做最近一次删除/清空操作。

- 删除后汇总数据实时更新。
- 撤销恢复文件和已经生成的结果；重做会再次应用该删除操作。

### US-04 下载

作为用户，我可以下载单个优化结果，或将所有完成结果打包为 ZIP 下载。

- 单文件沿用原文件名，在扩展名前添加 `.min`。
- 批量 ZIP 名称为 `PNGKing-optimized.zip`。
- 失败或未完成的项目不进入 ZIP。

### US-05 响应式与可访问性

- 适配 360px 以上移动端与桌面端。
- 所有核心操作可用键盘访问。
- 拖放区和状态变化包含可读文本，不仅依赖颜色。

### US-06 自定义压缩

作为用户，我可以选择压缩强度，或指定每张图片压缩后的目标文件大小。

- 压缩强度范围为 1–9，默认值为 6；目标大小启用时暂时禁用强度设置。
- 等待或处理中的图片显示预计压缩后体积，顶部汇总同步使用预估值。
- 目标大小提供 50 KB、100 KB、250 KB、500 KB、1 MB 和 2 MB。
- 目标模式选择不超过目标且与目标差距最小的候选，而不是体积最小的候选。
- 若所有候选均大于目标，则不生成结果、停止该图片的压缩，并在列表及全局消息中提醒用户。

## 4. 路由

- `/`：导入落地页。
- `/editor`：处理队列；空队列时展示空状态并可返回导入。
- 其他路径重定向到 `/`。

## 5. 非功能要求

- Vue 3 + Vite + TypeScript strict。
- Pinia 管理文件、结果、排序与撤销记录。
- Vue Router 4 管理页面。
- Element Plus 自动导入，主题变量位于 `src/styles/theme.css`。
- VueUse 提供文件选择与拖放能力。
- GitHub Actions 自动执行 lint、测试、类型检查、构建并部署 GitHub Pages。
